# Tech Debt Audit — context-visualizer

Repo state at audit: branch `main`, two commits ahead of `origin/main` (`59659d8`, `a0515ee`), Phase 4/5 artifacts for slice 127 uncommitted, no implementation started for slice 127.

## Executive summary

- The largest single debt concentration is **[project-structure-viz.jsx](../../../project-structure-viz.jsx)** (2110 LOC, 48 commits in last six months) — `ProjectPanel` alone is 605 LOC (lines 1125–1729), the textbook god-component smell, mixing eight independent handlers, drag logic, localStorage, and discovery UI in one function.
- **Slice 127 has not landed but its target code is dead-on-arrival.** Roughly 280 LOC of `serve.py` — `_handle_refresh_local`, `_handle_add_project`, `_handle_remove_project`, `_handle_list_projects`, the manifest helpers, the local-mode fallback in `_handle_info` — are being deleted. Aggressive cleanup here is now a *task-list completion* problem rather than a design problem. Flagged here so the audit reflects the *post-slice-127* shape the system is targeting.
- **Cross-language constant drift**: the panel-color palette is duplicated verbatim in [serve.py:507](../../../serve.py#L507) and [project-structure-viz.jsx:1123](../../../project-structure-viz.jsx#L1123). A change to one will silently desynchronize dashboard tiles from panel rows. CLAUDE.md explicitly forbids this pattern.
- **A slug-derivation idiom (`name.lower().replace(" ", "-")`) is open-coded in seven places** across [serve.py](../../../serve.py) (lines 40, 151, 159, 362, 647) and [parse.py](../../../parse.py) (lines 622, 643). One missed migration to MCP-supplied display names will cause project-key mismatches in only some endpoints.
- **Three nearly identical MCP `project_list` name→ID resolution blocks** at [serve.py:395–409](../../../serve.py#L395-L409), [serve.py:447–460](../../../serve.py#L447-L460), and [serve.py:524–535](../../../serve.py#L524-L535). The third variant differs subtly (skips the 404 path), which is itself a smell.
- **133 raw hex literals in the JSX** (out of ~200 places `THEME` is referenced) — the explicit "centralize palette" principle in [theme.js](../../../theme.js) covers only `--status-*` tokens. Most card backgrounds, borders, and muted text colors remain inline (`#1A1A2E`, `#2A2A4E`, `#8888AA`, `#6666AA`, `#555577`).
- **Three `setTimeout` calls without React cleanup** ([project-structure-viz.jsx:1179, 1280, 2031](../../../project-structure-viz.jsx#L1179)) fire in async handlers, then `setState` after the 3s window. If the user navigates between MCP/local modes or unmounts the panel during that window, React will warn about setState on an unmounted component.
- **`mcp_client.py` is single-threaded by lock, but the HTTP server is `HTTPServer` (single-threaded by default)** — so the lock is currently a coincidence rather than a guarantee. If anyone ever wraps `serve.py` in `ThreadingHTTPServer`, the dashboard handler ([serve.py:484](../../../serve.py#L484)) fires four sequential MCP calls per project and is the obvious place for serialization to hurt — `tools/call` IDs are generated under lock but reads from stdout are not multiplexed by id, so concurrent calls could read each other's responses.
- **`_extract_tool_result` ([mcp_client.py:243](../../../mcp_client.py#L243)) silently degrades MCP errors to dicts the caller can't distinguish from success.** If the server returns `{isError: true, content: [...]}`, the client unwraps the text content and returns a dict; only `_handle_structures` checks `model.get("isError")` ([serve.py:357](../../../serve.py#L357)). The other six MCP-using handlers do not check `isError` at all.
- **README ([README.md:7-11](../../../README.md#L7-L11)) documents architecture slice 127 is removing.** Line 8 promises local fallback that 127 deletes; line 10 promises add/remove/discover UI that 127 deletes; line 21 says `python serve.py` but the project requires `uv run python serve.py` (no `python` on PATH after a fresh `uv sync`). User flagged this confusion during this session; no edit was made.

## Architectural mental model

The system is a single-host vanilla-JS dashboard that aggregates Context Forge project state. The backend is a `stdlib-only` Python HTTP server ([serve.py](../../../serve.py)) running on one port, with two data sources: a long-lived MCP subprocess speaking JSON-RPC over stdio ([mcp_client.py](../../../mcp_client.py)), or a local filesystem scanner ([parse.py](../../../parse.py)) that emits the same JSON shape. The frontend is a single 2.1k-line JSX file Babel-transpiled in the browser, mounted into [index.html](../../../index.html) with no build step.

The state model is **MCP-as-truth + local-prefs side channel**. The MCP server owns the project catalog and structure; the visualizer owns starred/hidden flags via a separate `~/.config/context-visualizer/project-prefs.json` file written by `PATCH /api/projects/{key}` ([serve.py:683](../../../serve.py#L683)). Slice 127 is currently removing the last vestiges of the local-only path — the `projects/manifest.json` catalog, the add/remove/discover endpoints, and the local fallback in `/api/structures`. Once 127 lands, `parse.py` survives only as an importable library for `/api/discover` (the "what projects exist on this disk that I don't know about yet?" entry point); `serve.py` shrinks to a thin MCP proxy with a prefs side-table and a dashboard aggregator. That target shape is **good**: it eliminates two-source-of-truth bugs at the cost of forcing MCP to be the integration seam. Most findings in this audit are about how the current code is sized for the *old* architecture and should be reshaped while slice 127 is in flight.

## Things that look bad but are actually fine

- **2110-LOC JSX file with no build step.** Loud red flag against React best practice — but the project's headline feature ([README.md:11](../../../README.md#L11)) is "zero dependencies, vanilla JS frontend with in-browser JSX transform." Splitting into multiple files is a valid follow-up (F002) but the *no-build-step* property is intentional and load-bearing for the project's positioning. Don't introduce webpack/vite to "fix" this.
- **`from __future__ import annotations` at [serve.py:12](../../../serve.py#L12) on a Python 3.13 codebase.** The Python rules at `.claude/rules/python.md` explicitly say *keep this for 3.12/3.13*. Verified intentional; leave.
- **`HTTPServer` (single-threaded), not `ThreadingHTTPServer`.** Looks like a performance miss, but `mcp_client.py` is intentionally single-threaded over stdio (one process, one stdin, one stdout). Switching to threading would require either serializing all MCP traffic through a queue or instantiating one MCP child per thread, both of which add complexity without measurable benefit for a single-user local dashboard.
- **`time.monotonic()` polling loop in `_read_response` ([mcp_client.py:185-212](../../../mcp_client.py#L185-L212))** instead of `asyncio`. Looks dated. But the entire server is sync `http.server`, and the MCP transport is line-delimited JSON-RPC. An `asyncio` retrofit would touch every handler in `serve.py`. The polling loop is tight (100ms inner timeout), no measurable wall-clock cost on local stdio. Leave it.
- **Hand-rolled YAML frontmatter parser ([parse.py:61-80](../../../parse.py#L61-L80))** rather than PyYAML. Documented intentional (zero-dependency principle). Tests cover the cases that actually appear in the codebase. F022 covers what to add; this isn't a defect.
- **`.claude/rules/react.md` says "Use Tailwind 4 + Radix + Lucide; build with pnpm"** but none of those are installed. The rules file is a *global* preference for greenfield projects, not a description of this stack. The vanilla-JS-with-in-browser-Babel decision is explicit. No finding.
- **Local `.env` with `OPENROUTER_API_KEY` and `SQUADRON_APP_NAME`.** `.env` is in `.gitignore` ([.gitignore:30](../../../.gitignore#L30)), `git ls-files` confirms not tracked, and no code in this repo reads either variable (only `os.environ` reference is at [mcp_client.py:69](../../../mcp_client.py#L69) for child-process env passthrough). Orphaned from `squadron`. Not a leak; not a finding.
- **Cache-buster `?v=${Date.now()}` on JSX fetch.** Looks like a "we can't figure out caching" smell. It's actually deliberate from slice 121 (`feat: add slice info panel and cache-busting improvements`). For a local dev dashboard with no build, this is correct.
- **`@pytest.mark.skipif(not CF_PATH.exists())` at multiple sites.** Looks like the test suite is hiding failures. The skipped tests are integration tests that *require* a sibling `context-forge` checkout to be meaningful. Skip-with-reason is the right call. (F016 covers the documentation gap, not the skip itself.)
